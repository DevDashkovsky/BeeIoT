Метрики: Код ответа (должен быть успешным) + Время отклика


=== 1. РЕГИСТРАЦИЯ ПОЛЬЗОВАТЕЛЯ ===

Content-Type: application/json
Body:
{
  "email": "loadtest{N}@example.com",
  "password": "Test1234567890"
}
Ожидаемый код: 200
Проверка: status == 200, response_time < 2000ms

====== СЦЕНАРИИ НАГРУЗКИ ======

СЦЕНАРИЙ 1: Базовый поток (1 пользователь)
Шаги: 1 → 2 → 3 → 4 → 5 → 6 → 16
Цель: Проверка базовой функциональности

СЦЕНАРИЙ 2: Полный цикл (1 пользователь)
Шаги: 1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 11 → 12 → 14 → 15 → 16
Цель: Проверка всех функций

СЦЕНАРИЙ 4: Чтение данных (авторизованный пользователь)
Повторять: 5 → 6 → 8 (100-1000 раз)
Цель: Тестирование read-операций

СЦЕНАРИЙ 5: Создание/удаление (авторизованный пользователь)
Повторять: 4 → 5 → 15 (50-500 раз)
Цель: Тестирование write-операций

Коды успеха:
- 200: Успешный запрос
- 500: Ошибка сервера (отслеживать количество)
- 401/403: Ошибки авторизации (не должно быть при корректных токенах)


====== НАГРУЗКА =======

Должен выдерживать до 125 пользователей одновременно при кол-ве запросов до 30 в секунду одновременно

====== ЗАМЕЧАНИЯ ======

{N} - заменять на уникальный номер пользователя (1, 2, 3, ...)
{token} - JWT токен из ответа авторизации
Коды подтверждения (123456) - в реальности придут на email, но для теста их надо замокать.
Финальные коды ответа могут меняться, поэтому следует проверять не код 200, а код 2xx (где x - любая цифра)
