# BeeIoT: Интеграционные тесты прошивки

Данная директория содержит набор инструментов для проверки работоспособности прошивки в реальных (или полу-реальных) условиях. Поскольку прошивка работает на MicroPython через аппаратный GSM-модуль, полноценное Unit-тестирование на локальной машине без эмуляторов оборудования затруднено.

Поэтому здесь реализован **Интеграционный Тестовый Сервер**.

## Описание
Скрипт `test_server.py` эмулирует сторону Backend-сервера. Он:
1. Подключается к указанному MQTT брокеру.
2. Подписывается на топики получения телеметрии (`data` и `status`).
3. Валидирует входящие JSON-структуры на соответствие документации (папка `docs/`).
4. Автоматически отправляет тестовую конфигурацию (`config`) после получения статуса, чтобы проверить, как устройство обрабатывает состояние `LISTEN_CONFIG`.

## Требования
Убедитесь, что у вас установлен Python 3.7+.

1. Создайте виртуальное окружение:
   ```bash
   python3 -m venv venv
   source venv/bin/activate  # Для Linux/macOS
   # venv\Scripts\activate   # Для Windows
   ```
2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Как проводить тест

1. **Подготовьте устройство (ESP32):**
   - Убедитесь, что устройство подключено к SIM-карте с доступом в интернет.
   - В `config.py` прошивки задайте `SENSOR_ID = "beeiot_001"` (или измените ID в `test_server.py`).
   - Залейте прошивку в микроконтроллер.
   
2. **Запустите тестовый сервер на ПК:**
   ```bash
   python test_server.py
   ```

3. **Включите устройство:**
   - ESP32 включится, прочитает датчики температуры и шума.
   - Поднимет GPRS-соединение с помощью SIM800L.
   - Отправит данные на брокер.
   - В консоли ПК вы должны увидеть, что сервер успешно прочитал данные, валидировал все ключи и вывел `[PASS]`.
   - Сервер отправит команду `config` назад на устройство. Спустя пару секунд (до перехода ESP32 в Deep Sleep), в UART логах с ESP32 мы должны увидеть применение нового конфига.
