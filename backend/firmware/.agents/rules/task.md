МАСТЕР ПРОМПТ (ОБЩЕЕ ТЕХНИЧЕСКОЕ ЗАДАНИЕ ДЛЯ ВСЕХ АГЕНТОВ)

ПРОЕКТ BEEIOT: ПРОШИВКА ДЛЯ IOT-ДАТЧИКА НА ESP32
Цель: Разработка энергоэффективной и отказоустойчивой прошивки для сбора данных и отправки по MQTT через GPRS.
Язык: MicroPython
Оборудование: Микроконтроллер ESP32, GSM-модуль SIM800L (связь по UART, AT-команды), датчик температуры DS18B20 (1-Wire), датчик шума KY-038 (чтение через ADC).

ПРАВИЛА РАБОТЫ С ДОКУМЕНТАЦИЕЙ (ПАПКА DOCS)
В папке docs лежат файлы с описанием распиновки, адресами серверов и структурой MQTT-топиков. Эти файлы нужны исключительно для извлечения информации. Запрещено копировать файлы из docs в итоговый код прошивки. Все константы из docs (пины, URL) должны быть перенесены в конфигурационный файл config.py. Сам ID датчика (sensor_id) будет зашиваться вручную пользователем в этот же config.py перед прошивкой.

ЖИЗНЕННЫЙ ЦИКЛ УСТРОЙСТВА (АЛГОРИТМ РАБОТЫ)

Пробуждение ESP32 по таймеру из режима Deep Sleep.

Инициализация датчиков и чтение данных. Если датчик не ответил или вернул ошибку, его значение приравнивается к -1.

Включение питания модуля SIM800L и ожидание регистрации в сети GPRS.

Если сеть недоступна: сохранение данных во flash-память ESP32 (описание ниже) и немедленный переход к шагу 8.

Подключение к MQTT-брокеру. Публикация накопившихся в памяти данных, затем публикация новых данных в топик /device/sensor_id/data и статуса в топик /device/sensor_id/status.

Подписка на топик /device/sensor_id/config для получения команд от сервера. Ожидание в течение 5 секунд.

Если пришла конфигурация — применение и сохранение настроек.

Жесткое выключение модуля SIM800L (AT+CPOWD=1 или отключение питания) и перевод ESP32 в режим Deep Sleep до следующего цикла.

ЛОКАЛЬНОЕ ХРАНИЛИЩЕ (ОФЛАЙН РЕЖИМ)
При отсутствии сети JSON-данные должны сохраняться локально во flash-память. Необходимо реализовать механизм кольцевого буфера. Выделить фиксированный лимит памяти под файл (например, 10 КБ). Если файл заполнен, новые данные должны перезаписывать самые старые записи. Важна актуальность данных, а не их историческая полнота.

ТРЕБОВАНИЯ К КОДУ
Писать чистый, модульный код. Запрещены бесконечные циклы while без таймаутов, особенно при работе с сетью и UART. При любых критических сбоях система должна уходить в Deep Sleep, чтобы не разрядить батарею.
