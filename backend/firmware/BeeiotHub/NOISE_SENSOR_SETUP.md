## 1. Подключение датчика

### Схема подключения KY-038 к ESP32:

```
KY-038          ESP32
------          -----
VCC      →      3.3V или 5V (зависит от модуля)
GND      →      GND
AO       →      GPIO 6 (ADC1_CH5) - аналоговый выход
DO       →      GPIO 7 (опционально) - цифровой выход
```

### ⚠️ Важные замечания:
- **AO (аналоговый выход)** - обязательно подключите к ADC-совместимому пину
- **DO (цифровой выход)** - опционально, срабатывает при превышении порога
- Используйте только пины **ADC1** (GPIO 32-39, 36, 37, 38, 39 или GPIO 0-10 в зависимости от ESP32)
- **НЕ используйте ADC2** (GPIO 0, 2, 4, 12-15, 25-27) если работает WiFi!

### Рекомендуемые пины для ESP32-S3:
- **GPIO 1-10** - ADC1, безопасны для использования
- Текущая настройка: **GPIO 6** (аналоговый), **GPIO 7** (цифровой)

### Если нужно изменить пины:
1. **KY038_ANALOG_PIN** - пин для аналогового сигнала (AO)
2. **KY038_DIGITAL_PIN** - пин для цифрового сигнала (DO), можно оставить `None` если не используете
3. **DB_CALIBRATION** - калибровочное смещение в децибелах (по умолчанию 65.0)

## 3. Калибровка датчика

### 📊 Метод калибровки (Peak-to-Peak)

Проект использует современный метод измерения шума через анализ размаха сигнала:

1. **Сбор данных**: В течение 50мс собираются максимальное и минимальное значения сигнала
2. **Вычисление размаха**: `peak_to_peak = signal_max - signal_min`
3. **Конвертация в Вольты**: `volts = (peak_to_peak * 3.3) / 65535`
4. **Расчёт дБ**: `db = 20 * log10(volts) + DB_CALIBRATION`

### Как откалибровать DB_CALIBRATION:

1. **Используйте приложение для измерения дБ на смартфоне** (например, "Sound Meter")
2. **Запустите test_sensors.py** или **test_quet.py**
3. **Сравните показания** датчика и приложения
4. **Скорректируйте DB_CALIBRATION**:
   - Если датчик показывает **меньше** чем приложение → **увеличьте** DB_CALIBRATION
   - Если датчик показывает **больше** чем приложение → **уменьшите** DB_CALIBRATION

**Пример:**
- Приложение показывает: 70 дБ
- Датчик показывает: 65 дБ
- Нужно увеличить DB_CALIBRATION на 5 (с 65.0 до 70.0)

## 4. Метод измерения

### Код из test_quet.py (используется в sensors.py):

```python
def get_loudness(window_ms=50):
    """Измерение громкости через peak-to-peak анализ"""
    start_time = time.ticks_ms()
    
    signal_max = 0
    signal_min = 65535  # 16-битное чтение
    
    # Сбор выборок в течение 50мс
    while time.ticks_diff(time.ticks_ms(), start_time) < window_ms:
        sample = adc.read_u16()  # Читаем 16-бит (0-65535)
        
        if sample > signal_max:
            signal_max = sample
        if sample < signal_min:
            signal_min = sample
    
    # Амплитуда сигнала
    peak_to_peak = signal_max - signal_min
    
    # Конвертация в Вольты
    volts = (peak_to_peak * 3.3) / 65535
    
    return volts

# Расчёт дБ
volts = get_loudness()
if volts < 0.005:
    volts = 0.005  # Защита от log(0)
db = 20 * math.log10(volts) + DB_CALIBRATION
```

### Преимущества этого метода:
✅ **Точнее** - анализирует динамический диапазон сигнала
✅ **Быстрее** - 50мс вместо нескольких секунд усреднения
✅ **Физически корректен** - использует логарифмическую шкалу
✅ **Реагирует на изменения** - видит пики шума

## 6. Настройка параметров

### В config.py доступны следующие параметры:

```python
# ===== GPIO пины =====
KY038_ANALOG_PIN = 6  # Аналоговый вход (AO)
KY038_DIGITAL_PIN = 7  # Цифровой выход (DO)

# ===== Калибровка =====
DB_CALIBRATION = 65.0  # Калибровочное смещение в дБ

# ===== Частота опроса =====
DEFAULT_SAMPLING_NOISE = 60  # Как часто читать датчик (секунды)
```

### Настройка порога цифрового выхода:

На датчике KY-038 есть **потенциометр** (винт):
- 🔄 По часовой стрелке → порог выше (менее чувствительный)
- 🔄 Против часовой → порог ниже (более чувствительный)
- 💡 Светодиод загорается при превышении порога

## ⚙️ Технические детали

### Характеристики KY-038:
- **Питание:** 3.3V - 5V
- **Чувствительность:** Регулируется потенциометром
- **Частотный диапазон:** 100Hz - 10kHz
- **Аналоговый выход:** 0 - 3.3V
- **Цифровой выход:** LOW при превышении порога

### Параметры измерения:
- **Окно выборки:** 50 мс
- **Разрядность АЦП:** 16 бит (0-65535)
- **Диапазон напряжения:** 0 - 3.3V (ATTN_11DB)
- **Формула:** `dB = 20 * log10(V) + calibration`

## 🔧 Решение проблем

**Проблема:** Значения всегда ~40 дБ и не меняются
- ✅ Проверьте подключение аналогового выхода (AO)
- ✅ Убедитесь, что микрофон не закрыт
- ✅ Попробуйте другой GPIO пин

**Проблема:** Значения слишком высокие/низкие
- ✅ Скорректируйте DB_CALIBRATION в config.py
- ✅ Сравните с приложением для измерения дБ

**Проблема:** Значения хаотично прыгают
- ✅ Это нормально для микрофона!
- ✅ Звук - это динамический сигнал
- ✅ Увеличьте window_ms для сглаживания
