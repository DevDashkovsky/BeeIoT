# Результаты нагрузочного тестирования BeeIoT API

## 1. Общая информация

| Параметр | Значение |
|----------|----------|
| **Проект** | BeeIoT Backend API |
| **Дата тестирования** | 13 февраля 2026 г. |
| **Инструмент** | k6 (Grafana) |
| **Окружение** | Docker |

## 2. Цель тестирования

Проверить, что API выдерживает целевую нагрузку без деградации производительности.

**Критерии успешности:**
- 95-й перцентиль времени ответа < 2000 мс
- Процент ошибок < 1%
- Стабильная работа под нагрузкой около ста одновременных пользователей
- Все эндпоинты возвращают корректные ответы

## 4. Сценарий нагрузки

### Тип тестирования

Load Testing + Stress Testing + Spike Testing

### Профиль нагрузки

Тест состоял из 4 параллельных сценариев:

#### 4.1. Basic Flow (Базовая нагрузка с рампингом)

| Параметр | Значение |
|----------|----------|
| **Тип** | Ramp-up/Ramp-down |
| **VUs** | 0 → 100 (5 стадий) |
| **Длительность** | 3 минуты 30 секунд |
| **Graceful stop** | 30 секунд |
| **Сценарий** | Регистрация → Вход → Создание улья → Удаление |

#### 4.2. Full Cycle (Полный жизненный цикл)

| Параметр | Значение |
|----------|----------|
| **Тип** | Per-VU iterations |
| **VUs** | 10 |
| **Итераций** | 1 на пользователя (всего 10) |
| **Старт** | +10 секунд от начала |
| **Сценарий** | Регистрация → Создание → Обновление → Смена пароля → Удаление |

#### 4.3. Read Heavy (Операции чтения)

| Параметр | Значение |
|----------|----------|
| **Тип** | Constant VUs |
| **VUs** | 20 |
| **Длительность** | 2 минуты |
| **Старт** | +2 минуты от начала |
| **Сценарий** | Циклические операции: List Hives → Get Hive → Calc Queen (10 итераций) |

#### 4.4. Write Heavy (Операции записи)

| Параметр | Значение |
|----------|----------|
| **Тип** | Constant VUs |
| **VUs** | 5 |
| **Длительность** | 2 минуты |
| **Старт** | +2 минуты от начала |
| **Сценарий** | Создание и обновление ульев |

## 6. Выводы

- Обработано 29,045 HTTP-запросов с нулевым процентом ошибок
- Все 69,405 проверок валидации прошли успешно (100%)
- 95-й перцентиль времени ответа (420 мс) в 5 раз ниже установленного порога (2000 мс)
- Система стабильно работала под нагрузкой 125 одновременных пользователей
- Пропускная способность ~45 req/s достаточна для текущих требований
- узких мест не обнаружено

```bash
# Запуск нагрузочных тестов
make load_test

# Пересборка и тестирование
make load_test_build

# Остановка инфраструктуры
make stop
```
